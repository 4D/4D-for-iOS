Now it's time to create your iOS detail form interface with Xcode!

Here's the result we want to achieve:

For each cell, we're going to add a:

  • Profile image Header
  • A field that will be duplicated

Open the storyboard file with Xcode

First, open your storyboard file in Xcode.

It's quite empty, so let's add some content!

Add a Scroll View

For our detail form to be scrollable, the first step is to drag and drop an Scroll View from the Object library.

From the Size inspector panel, set the Scroll View Width value to 359 and the Height value to 667. Then set the X value to 8 and the Y value to 0.

Next, add 4 constraint by clicking on the Add New Constraints button (Trailing, Leading, Top and Bottom) as shown on the following visual:

Add a Vertical Stack View

Drag and drop a Vertical Stack View from the Object library in Scroll View and From the Size inspector panel, set the Width value to 359 and the Height value to 202. Then set the X value to 0 and the Y value to 8.

As the Stack View width and height are defined by elements inside it, we will configure size and contraints later.

Add Views to the Stack Views

Drag and drop a view from the Object library to the Stack View you've just added to your storyboard. let's name it Field View 1.

Then drag and drop another view from the Object library directly in the Document Outline at the same level at the same level as Field View 1 to include it in the Stack View and name it Field View 2.

Then to add space between views, select your Stack View, go to the Attributes inspector and set the spacing value to 8.

You are done with the Stack View constraints and spacing !

Select both Field View 1 and Field View 2 and from the Size inspector panel, set Width value to 359 and the Height value to 97.

Select Field View 1 and add the following constraints :

For the Stack View to have the same width as the scrollview you need to add a contraint :

  • Press Ctrl and drag the Stack View to the Scroll View. When you release the mouse button, the Interface Builder displays a menu with a list of possible constraints. Choose Equals Width !

Now your Stack View will be the same width as the Scroll View.

Next, with the Stack View selected, add 4 constraint by clicking on the Add New Constraints button (Trailing, Leading, Top and Bottom) as shown on the following visual:

Here what you see when constraints have been added :

Now that you have now 2 views in your Stack View and you defined constraints, Let's focus on Field View 1.

Add an Image View to Field View 1

From the Object library, drag and drop an Image View into Field View 1.

From the Size inspector panel, set the Image View Width value to 359 and the Height value to 97. Then set the X value to 0 and the Y value to 0.

Next, add a leading space constraint by clicking on the Add New Constraints button. Add width and height constraints too, as shown on the following visual:

From the Size inspector change the Intrinsic Size from Default to Placeholder

Add Labels to cell to Field View 2

Title label

From the Object library, drag and drop a Label into Field View 2 and set the Width value to 343 and the Height value to 22. Then set the X value to 8 and the Y value to 8.

Double click on the label and name it "___FIELD_LABEL___". Then from the attribute inspector set the font to Helvetica Neue Light 18.0 and the color to BackgroundColor:

Content label

Duplicate the Title label and set the Width value to 339 and the Height value to 44. Then set the X value to 12 and the Y value to 37.

Double click on the label and name it "<___FIELD_LABEL___>". Then from the attribute inspector set the font to Helvetica Neue Light 20.0 and the color to Black Color:

Select Content label and Title label and add 4 contraints by clicking on the Add New Constraints button as shown on the following visual:

You are finally done with graphical elements! All your constraints are well defined :)

How to get data into your cells

Image View

Select your Image View and go to Identity inspector > User Defined Runtime Attributes and click the plus button to add a row.

  • Key Path: Start with bindTo to activate binding on the component. Enter bindTo.record.___FIELD_1___
  • Type: Always String
  • Value: The attribute name. Enter ___FIELD_1_TYPE___

Labels

For the first label we are going to add several lines mainly to display icons:

You have also to define the Class as well as the Module as we can see on the following visual :

Select the second label and add a row in the Defined Runtime Attributes:

  • Key Path: bindTo.record.___FIELD___
  • Type: String
  • Value: ___FIELD_TYPE___

Add TAGS to the cell to be duplicated

Basically the Field View 2 includes elements that will be duplicated to have as many field as you defined in the project editor.

Each storyboard element has a Object ID. For example if you select the Field View 2 and go right to the Identify inspector you will it's Object ID :

So to tell the project editor that all elements included in this view will be duplicated we must add TAG to each element as well as for constraints.

So to do that it will be a little bit tricky... Let's begin opening the storyboard file with your favorite code editor !

Field View 2

  • Select the Field View 2 and get Object ID from the Identity inspector of your Storyboard. In my project, the Field View 2 ID is : I50-3M-sUG.

  • Search for it and replace by TAG-FD-001 each time it appears in the storyboard xml code. As you can see this Object ID is mentionned several times so be careful not to forget any Object ID.

Save the storyboard xml file. The Field View 2 Object ID has changed in your Storyboard in Xcode :

First label

  • Now get the first label Object ID from the Identity inspector of your Storyboard and replace it by TAG-FD-002 each time it appears in the storyboard xml code.

Second label

  • Finally get the second label Oject ID from the Identity inspector of your Storyboard and replace it by TAG-FD-003 each time it appears in the storyboard xml code.

Constraints

Both labels have constraints that we need to tag as well :

As for View Field 2 and label get all 7 Constraints Object ID from the Identity inspector of your Storyboard and replace it by TAG-FD-004, TAG-FD-005, TAG-FD-006, TAG-FD-007, TAG-FD-008, TAG-FD-009 and TAG-FD-010.

Go straight to your project editor select your Custom detail template from the Forms section and Build.

Pretty good for a first try :)

The Field View 2 is well duplicated ! Let's see if we can make this app a little bit more sexy!

Customize your app

We are going to add corner radius to the Image View as well as the Field View 2.

Select your Image View and add the two following lines in the User Defined Runtime Attributes :

  • Key Path: cornerRadius
  • Type: Number
  • Value: 12

and

  • Key Path: layer.masksToBounds
  • Type: Boolean
  • Value: Check the box

In this tutorial we will cover many aspects of the template creation. We will guide you through the creation steps of List form that includes a searchBar, a sectionfield and a Table with an image, a title and a subtitle for each cell.

Download our Starter Project, which contains a template initial folder

CUSTOM LISTFORM STARTER PROJECT

Lets create our first Listform template. In this starter project, you will find : an icon that will be displayed in the project editor when you select your custom template : layoutIconx2.png icon in 160x160px a manifest file that includes a basic description of the template a template svg file that will be the representation of your template that will be display when you define your listForm content and where you will drag and drop your fields Source folder that includes the storyboard (graphical interface) and swift file (code for the form)

Create a …/Resources/mobile/form/list folder and drag and drop the Custom List Template in it.

What are those files and what it is used for and how to build it ?

Template icon

You can create your own template icon. This icon must be 160x160px and have to be named “layoutIcon.png” This icon will be used in the project editor in the template selector window when you select a table in the form section as well as on top of table name when the template has been selected.

If this icon is missing in your custom template folder the template.svg file (that we are going to study later) will be displayed in the template selector window and a icon with a question mark will be displayed on top of each table that have a missing custom template icon.

manifest.json file

The manifest file includes several informations about your custom template like: The storyboard type (listform/detailform) The template name * Fields number per cell

We will have 3 fields in each cell in this template: An profile image A Title * A subtitle

{
  "type": "listform",
  "name": "Custom list form",
  "fields": {
    "count": 3
  }
}

template.svg file

The template.svg file is a basic template representation. In this svg file you will have to define areas for you to be able to add fields to your custom list form from the project editor.

Here what we want to get :

Open this file with your favorite code editor.

Let’s focus on the different parts of your svg file and what you need to edit.

1. Title

Add your template title here.

2. ios:values

Inclues ids that will define your form areas : - search id refers to search field area : will allow you to drag and drop a field that will be the search criteria in your list form (optional) - section id that refers to the section field area : will allow you drag and drop a field that will be the sort criteria in your list form (optional) - f1, f2 and f3 ids refer to fields to display in each cell of your list form : will allow you to drag and drop field that will appear in you list form cell

3. Areas position, height and width

You will be able to define position, height and width as well as icons for: -Searchfield -Sectionfield -General fields that will be displayed in each table cells

  2. Area blue background position, height and width

Now that you have an icon, a basic template description and your svg file let’s focus work on Xcode !

4. Storyboard file

Here what we want to get

As you can see, we are going to add for each cell: an image a title * and a subtitle

Add ojects into your cell

Open the your Storyboard file into Xcode.

It is quite empty so let’s add our image and labels.

Add your Image View

From the Object library, drag and drop an Immage View into your cell. You type Image View into the search bar in the bottom to filter all assets.

From the Size inspector pane, put the Image View Width value to 110 and the Height value to 110. Then put the X value to 8 and Y value to 3.

Add a leading space constraint by clicking on the Add New Constraints button as well as width and height constraints

finally click on the Align button (at the bottom of the Interface Builder window) and check the Vertically in Container checkbox.

Your Image View is then well positionned !

Let’s work now on cell’s labels.

Title and subtitle labels

Let’s begin adding a view that will contain both title and subtitle labels. Search for View into the Object library and drag and drop it in the cell container.

From the Size inspector pane, put the View Width value to 277 and the Height value to 94. Then put the X value to 126 and Y value to 10.

As for the image view add contraints for the view to be well positionned.

Drag and drop a label into the view you just added from the Object Library

From the Size inspector pane, put the Label View Width value to 269 and the Height value to 32. Then put the X value to 8 and Y value to 8.

Duplicate the label and from the Size inspector pane, put the X value to 8 and Y value to 48 to the duplicate label.

Select both labels and add the following constraints

With a label seletected, double click on the Height constraint to edit it

Change the relation from Equal to Greater Than or Equal to have a variable label height (to deal with multiline labels).

Do the same with the second label.

How to get your data in your cells

Select your Image View and go to the Identity inspector under the User Defined Runtime Attributes and clic the plus button to add a row.

Key Path: start with bindTo to activate binding on the component so enter bindTo.record.FIELD1 Type: is always String (because value is here the attribute name of the ui widget) Value: is the attribute name so enter FIELD1_TYPE
